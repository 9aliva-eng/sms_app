# SMS Рассылка — Android App

Приложение для массовой SMS рассылки через реальные SIM карты.  
Поддерживает до 6 телефонов с 2 SIM каждый, работающих одновременно.  
База получателей и настройки хранятся в Google Sheets.

---

## 1. Установка Android Studio

1. Скачайте Android Studio с официального сайта: https://developer.android.com/studio
2. Запустите установщик и следуйте инструкциям (Windows/macOS/Linux).
3. При первом запуске установите:
   - Android SDK (API 34)
   - Android SDK Build-Tools
   - Android Emulator (опционально)
4. Убедитесь, что Java JDK установлена (Android Studio включает OpenJDK автоматически).

---

## 2. Открытие проекта в Android Studio

1. Запустите Android Studio.
2. Выберите **"Open"** (или File → Open).
3. Укажите папку `sms_app/` (корень проекта, содержащий `settings.gradle`).
4. Android Studio автоматически синхронизирует Gradle и скачает зависимости.  
   Дождитесь окончания синхронизации (прогресс-бар внизу).
5. Если появляется ошибка **"Gradle sync failed"**, проверьте подключение к интернету — скачиваются зависимости из Maven Central.

---

## 3. Настройка Google Sheets API

### 3.1 Создание проекта в Google Cloud Console

1. Перейдите в [Google Cloud Console](https://console.cloud.google.com/).
2. Создайте новый проект (или выберите существующий).
3. В меню **APIs & Services → Library** найдите **"Google Sheets API"** и нажмите **Enable**.

### 3.2 Создание API-ключа

1. Перейдите в **APIs & Services → Credentials**.
2. Нажмите **"+ Create Credentials"** → **"API Key"**.
3. Скопируйте созданный ключ.
4. **Опционально (рекомендуется):** Ограничьте ключ — в настройках ключа выберите  
   **API restrictions → Restrict key → Google Sheets API**.

> **Важно:** API-ключ обеспечивает только чтение публичных таблиц.  
> Для записи таблица должна быть открыта на редактирование для "всех у кого есть ссылка",  
> либо используйте OAuth 2.0 (см. ниже).

### 3.3 Получение ID таблицы

URL таблицы Google Sheets имеет вид:
```
https://docs.google.com/spreadsheets/d/SPREADSHEET_ID/edit
```
Скопируйте `SPREADSHEET_ID` — это и есть ID таблицы.

### 3.4 Настройка прав доступа к таблице

Для работы приложения:
- Таблица должна быть открыта на **просмотр** для "всех у кого есть ссылка" (чтение работает).
- Для **записи** через API-ключ — таблица должна быть открыта на **редактирование** всем.  
  (Более безопасный вариант — OAuth 2.0 Service Account, но требует дополнительной настройки.)

## 4. Структура Google Sheets таблицы

Создайте таблицу с тремя листами:

### Лист «Получатели»

| A (номер телефона) | B (статус)  | C (кто отправил) | D (время отправки) |
|--------------------|-------------|------------------|--------------------|
| 79991234567        | ожидает     |                  |                    |
| 79997654321        | ожидает     |                  |                    |

**Допустимые значения колонки B:**
- `ожидает` — номер ещё не обработан
- `в_работе` — телефон-агент взял в работу
- `отправлено` — SMS успешно отправлено
- `ошибка` — ошибка отправки

### Лист «Настройки»

| A (параметр)              | B (значение)        |
|---------------------------|---------------------|
| Текст СМС                 | Ваш текст сообщения |
| Начало рассылки           | 09:00               |
| Конец рассылки            | 21:00               |
| Часовой пояс              | UTC+9               |
| Интервал мин (сек)        | 180                 |
| Интервал макс (сек)       | 340                 |

> Строка 1 — заголовки (игнорируются приложением). Данные начинаются со строки 2.

### Лист «Агенты»

| A (имя телефона) | B (SIM1 номер) | C (SIM1 остаток) | D (SIM2 номер) | E (SIM2 остаток) | F (SIM1 отправлено) | G (SIM2 отправлено) | H (последняя активность) | I (статус агента) |
|------------------|----------------|------------------|----------------|------------------|---------------------|---------------------|--------------------------|-------------------|
| Phone1           | 79991112233    | 500              | 79994445566    | 500              | 0                   | 0                   |                          | активен           |

**Колонки C и E** (остаток СМС) заполняет **владелец вручную**.  
**Колонки F, G, H, I** — обновляет приложение автоматически.

**Допустимые значения колонки I:**
- `активен` — агент работает
- `остановлен` — агент остановлен (приложение-агент увидит этот флаг и остановится)

## 5. Сборка APK

### Через Android Studio (Debug APK):

1. Откройте меню **Build → Build Bundle(s) / APK(s) → Build APK(s)**.
2. После сборки появится уведомление **"APK(s) generated successfully"**.
3. Нажмите **"Locate"** чтобы открыть папку с APK.  
   Путь: `sms_app/app/build/outputs/apk/debug/app-debug.apk`

### Через Android Studio (Release APK):

1. **Build → Generate Signed Bundle / APK**.
2. Выберите **APK** → **Next**.
3. Создайте или выберите keystore (хранилище подписи).
4. Заполните данные подписи → **Next → Finish**.
5. APK находится в: `app/build/outputs/apk/release/app-release.apk`

### Через командную строку:

```bash
# В папке проекта sms_app/
./gradlew assembleDebug

# APK будет в:
# app/build/outputs/apk/debug/app-debug.apk
```

## 6. Установка APK на телефон

### Требования к телефону:
- Android 8.0 (API 26) или выше
- Две SIM карты (для режима АВТО)
- Разрешить установку из неизвестных источников

### Включение установки из неизвестных источников:
- **Android 8+:** Настройки → Приложения → (выберите браузер/менеджер файлов) → Установка неизвестных приложений → Разрешить

### Способы установки:

**Способ 1 — через USB:**
```bash
# Включите режим разработчика на телефоне:
# Настройки → О телефоне → Номер сборки (7 раз)
# Настройки → Для разработчиков → USB отладка → Включить

# Подключите телефон к ПК, затем:
adb install app/build/outputs/apk/debug/app-debug.apk
```

**Способ 2 — через файл:**
1. Скопируйте APK файл на телефон (через USB, облако, мессенджер).
2. Откройте APK файл на телефоне.
3. Подтвердите установку.

## 7. Первоначальная настройка приложения

1. Запустите приложение на каждом телефоне-агенте.
2. Приложение сразу откроет экран **Настройки** (если они ещё не заданы).
3. Заполните:
   - **Режим:** для агентов — АГЕНТ (переключатель выкл.), для телефона владельца — АДМИН
   - **Имя телефона:** уникальное имя (напр. `Phone1`, `Phone2` — должно совпадать с колонкой A листа «Агенты»)
   - **ID таблицы:** из URL Google Sheets
   - **API ключ:** ключ Google Sheets API
   - **Режим SIM:** АВТО (обе SIM) — рекомендуется
4. Нажмите **Сохранить настройки**.

### На телефонах-агентах:
- Нажмите **СТАРТ**
- Приложение начнёт работать в фоне
- Можно заблокировать экран

### На телефоне администратора (режим АДМИН):
- Нажмите **Обновить** для загрузки статуса всех агентов
- Кнопка **Остановить всё** установит статус `остановлен` для всех агентов в Sheets
- Агенты проверяют этот флаг при каждой итерации и остановятся

## 8. Защита от блокировки оператором

Приложение автоматически применяет несколько техник:

1. **Случайные интервалы** между отправками (настраивается в листе «Настройки»).
2. **Уникальный невидимый символ** (Zero-width space и подобные) вставляется в позицию 70 каждого SMS.
3. **Лимит 15 SMS в час** с одной SIM (жёстко прописан в коде).
4. **Время рассылки** ограничено окном, заданным в настройках (UTC+9).

## 9. Разрешения приложения

Приложение запрашивает следующие разрешения:

| Разрешение | Зачем |
|------------|-------|
| SEND_SMS | Отправка SMS |
| READ_PHONE_STATE | Определение SIM карт и subscriptionId |
| READ_PHONE_NUMBERS | Чтение номеров телефона |
| INTERNET | Работа с Google Sheets API |
| FOREGROUND_SERVICE | Фоновая работа |
| WAKE_LOCK | Не даёт процессору засыпать |
| RECEIVE_BOOT_COMPLETED | Автозапуск после перезагрузки |
| POST_NOTIFICATIONS | Показ уведомлений (Android 13+) |

## 10. Часто задаваемые вопросы

**Q: Приложение останавливается при заблокированном экране.**  
A: Убедитесь, что приложение добавлено в исключения оптимизации батареи.  
Настройки → Батарея → Оптимизация батареи → найдите "SMS Рассылка" → Не оптимизировать.

**Q: Ошибка "HTTP 403" при работе с Sheets.**  
A: Проверьте, что API ключ правильный и таблица открыта на редактирование.

**Q: SMS отправляются только с одной SIM.**  
A: Проверьте, что обе SIM активны и имеют положительный баланс в листе «Агенты».

**Q: Как добавить больше номеров получателей?**  
A: Просто добавьте строки в лист «Получатели» со статусом `ожидает`. Агенты автоматически возьмут новые номера в работу.
